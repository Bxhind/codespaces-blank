# Основной функционал

Приложение должно решать задачу учёта учебного процесса:

### 1. Управление студентами

У студента есть:

* Имя
* Фамилия
* Email
* Дата рождения
* Группа (строка)
* Список курсов, на которые он записан
* История посещений занятий

Пользователь должен иметь возможность:

* Добавлять студента
* Редактировать
* Удалять
* Смотреть карточку студента
* Смотреть список всех студентов с фильтром по группе

---

### 2. Управление курсами

Курс содержит:

* Название
* Преподавателя
* Количество учебных часов
* Количество кредитов
* Список студентов, которые записаны на курс

Пользователь может:

* Добавлять/редактировать/удалять курс
* Просматривать курс
* Добавлять студентов на курс

---

### 3. Посещаемость

Модель “Attendance” (Посещение):

* StudentId
* CourseId
* Дата
* Был ли студент присутствующим (bool)

Функционал:

* Отмечать посещения за конкретный день
* Просматривать посещаемость студента по курсу
* Просматривать сводную статистику курса (например, % посещаемости)

---

# Требования к архитектуре и БД

### 1. Использовать Entity Framework Code First

* Модели: Student, Course, Attendance
* Миграции (Add-Migration, Update-Database)
* Seed-данные (минимальный набор студентов/курсов)

### 2. Отношения

* Student — Attendance: один ко многим
* Course — Attendance: один ко многим
* Student — Course: многие ко многим

### 3. Модели ViewModel

Создать минимум 2 ViewModel, например:

* StudentDetailsViewModel — студент + его курсы + посещаемость
* CourseStatsViewModel — курс + список студентов + % посещаемости

### 4. Репозиторий + сервисный слой

Интерфейсы репозиториев:

* IStudentRepository
* ICourseRepository
* IAttendanceRepository

Сервисный слой:

* IStudentService
* ICourseService
* IAttendanceService

Всё внедряется через Dependency Injection.

# UI требования (кратко)

* Bootstrap для таблиц и форм
* Razor + HtmlHelper
* Форма добавления посещаемости должна быть удобной (например: список студентов + чекбоксы)
* Валидация на моделях (DataAnnotations)
* Валидация на клиенте

---

# Web API (минимум 2 эндпоинта)

Например:

* GET /api/courses/{id}/attendance — JSON статистики
* GET /api/students/search?name=... — поиск студентов
Все остальные по вашему желанию
